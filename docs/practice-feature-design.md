# 数学乐园"练一练"功能设计方案

为数学乐园的7个演示模块添加互动练习功能，帮助学生巩固所学知识。

---

## 一、设计理念

### 核心原则

1. **与演示紧密结合** - 练习题型直接对应演示内容
2. **动态题目生成** - 每次练习实时生成全新题目，拒绝枯燥重复
3. **即时反馈** - 答对/答错立即给予视觉和音效反馈
4. **渐进难度** - 从简单到复杂，逐步提升
5. **趣味激励** - 积分、星星、徽章等激励机制

### 交互模式

- **选择题** - 点选正确答案
- **拖拽题** - 拖动元素到正确位置
- **填空题** - 输入数字答案
- **匹配题** - 连线配对

## 四、UI设计要点

### 视觉风格

- 与现有演示页面风格保持一致
- 使用明亮、友好的配色
- 圆角设计，适合儿童使用
- 大按钮，易于触控

### 反馈动画

| 场景     | 动画效果               | 音效       |
| -------- | ---------------------- | ---------- |
| 答对     | 🎉 金色粒子 + 星星飞出 | 欢快叮咚声 |
| 答错     | 🔴 轻微抖动 + 红色闪烁 | 低沉提示音 |
| 完成练习 | 🏆 奖杯动画 + 结算页   | 胜利号角   |

_结算页操作：_

- **再练一次**：重置进度，保留当前题目重新练习
- **重新出题**：调用生成器，生成全新一组题目进行挑战

## 五、激励系统

### 即时奖励

### 评分机制

- **满分制**: 每组练习总分固定 100 分
- **单题得分**: 根据题目数量动态计算 (`ceil(100 / queston_count)`)
- **连击奖励**: 连续答对触发火焰/星星特效
- **完成一组**: 解锁成就徽章
